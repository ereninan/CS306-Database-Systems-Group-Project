## 📖 About the Project
This project involves developing a comprehensive database system to manage the daily operations and administrative activities of a growing gym network. 

Gym facilities handle a massive amount of information every single day, such as keeping track of their various branches, staff members, registered customers, and physical fitness machines. The main goal of this database is to collect all this fragmented information into one centralized and organized platform. By doing this, the gym management can easily see:
* Which trainer works at which branch,
* Where specific equipment is currently located,
* Which members are actively attending group workouts.

This centralized tracking not only makes the daily management of the gym much easier and safer, but it also helps in preventing data loss and scheduling conflicts.

## 🛠 Technologies
* **Backend & UI:** PHP
* **Database:** SQL (MySQL)

## 📂 Directory Structure
Our project consists of 4 main phases, reflecting the development steps followed throughout the semester:

* 📁 **Phase 1:** Requirement analysis, creation of entity-relationship (ER) diagrams, and conceptual database design.
* 📁 **Phase 2:** Conversion of conceptual design into relational schemas and normalization steps.
* 📁 **Phase 3:** Physical creation of the database (DDL), table creation, and writing basic SQL queries/data manipulation operations (DML).
* 📁 **Phase 4:** Development of the web-based user interface (UI) using PHP and integration with the database to complete the project.
